FaciliChat Design System

A base visual de uma comunicação que não falha em silêncio

Este é o sistema de design do FaciliChat: tokens, componentes e padrões prontos para construção. A interface é deliberadamente clean — branco, cinza e um único azul de destaque — porque ela hospeda a marca de cada cliente e nunca deve competir com ela.

01

Visão geral

Quatro princípios guiam cada decisão visual do FaciliChat. Eles existem porque o produto resolve um problema de confiança: o cliente que espera semanas por um serviço sem nenhuma comunicação. A interface precisa transmitir controle e resposta imediata sem nunca chamar atenção para si.

Princípio 01

Discrição que dá palco ao cliente

Cada conta carrega a logo de uma conservadora diferente. Fundos brancos e cinza-gelo, tipografia neutra e um único azul de acento garantem que a marca do cliente seja sempre o elemento colorido mais forte da tela.

Princípio 02

O chat é o palco, o ticket é o bastidor

A conversa é o que o usuário vê e sente. Os tickets são infraestrutura invisível. O sistema de design trata a bolha de mensagem como componente-herói e o status de ticket como informação calma, nunca alarmante.

Princípio 03

Clareza acima de ornamento

Linguagem simples, estados em português direto (Recebido, Em andamento, Agendado, Concluído) e zero jargão técnico. A IA narra fatos estruturados — nunca inventa prazos nem fala em nome de um supervisor.

Princípio 04

Consistência que se aprende uma vez

Sete perfis usam o mesmo vocabulário visual. Um botão azul significa a mesma coisa para o cliente e para o gestor. Tokens compartilhados fazem a interface inteira responder como um só corpo.

02

Marca & Logo

A logo é fixa. Não pode ser redesenhada, recolorida fora das versões aprovadas, distorcida ou reconstruída. As versões abaixo são as únicas válidas — use o arquivo vetorial correto para cada fundo.

Ativo travado — não modificar

Logo horizontal (assinatura principal)

Aplicação preferencial em cabeçalhos, e-mails e documentos. Mantenha a área de respiro mínima igual à altura do símbolo de cada lado.

Sobre branco · uso padrão
Sobre azul da marca · versão monocromática branca

Símbolo isolado

O balão com check funciona como ícone de app, favicon e avatar do sistema. Use somente quando o contexto já identifica o produto.

PODE

NÃO PODE

11

Chat & Tickets

O componente que define o produto. A conversa é o palco: bolhas estilo mensageiro, resposta imediata da IA, indicador de digitação. O ticket é o bastidor: aparece como status calmo, nunca como alarme. A IA narra apenas campos estruturados — nunca fala por um supervisor nem inventa prazos.

Bolhas de mensagem

Recebida (branca, à esquerda) · Enviada (azul, à direita) · Sistema/ticket (pílula central neutra).
